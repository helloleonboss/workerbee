# 方案

## 概述

通过点滴记录构建工作日志，支持日报、周报、月报、季报、年报的生成。记录片段由 agent 聚合整理后输出报告。

## 目录结构

```
~/.workerbee/
├── .workerbee.config.json  # 应用配置
├── logs/                   # 原始点滴记录
│   ├── 2026-04-13.md
│   └── ...
└── reports/                 # 生成的报告
    ├── 2026-04-13-日报.md
    └── 2026-04-周报.md
```

## 记录格式

每次记录追加到当日的 Markdown 文件：

```markdown
---
date: 2026-04-13
---

## 14:30

讨论了Q2规划，结论是...

## 15:00

写代码修复了登录bug
```

### 格式规范

- **文件命名**：`YYYY-MM-DD.md`，一个文件一天
- **时间锚点**：`## HH:mm` 作为每次记录的开始标记
- **无标签**：不预设分类，语义由 agent 理解
- **可补录**：支持记录过去某个时间点发生的事
- **原文优先**：保持口语化，agent 会处理格式

## 核心流程

1. 用户安装完成后，首次启动引导选择存储文件夹（默认 `~/.workerbee/`）
2. 配置文件保存到 `.workerbee.config.json`
3. 用户通过 `ctrl+shift+space` 呼出记录框
4. 记录内容按日期追加到 `logs/YYYY-MM-DD.md`
5. Agent 读取指定日期范围的 `.md` 文件，按时间线整理，输出报告到 `reports/`

## 记录框交互

### 核心原则

**快开、快写、快关**——摩擦越小，使用越频繁。

### 交互机制

| 机制                 | 说明                                |
| -------------------- | ----------------------------------- |
| **全局快捷键**       | `ctrl+shift+space` 唤出，支持自定义 |
| **ESC 关闭**         | 不想记了就按 ESC，关掉弹窗          |
| **时间默认当前**     | 自动填充当前时间，可手动修改补录    |
| **Enter 提交**       | 写完直接回车，不用找按钮            |
| **Shift+Enter 换行** | 需要多段时换行，不至于一条走天下    |
| **输入框自动聚焦**   | 唤出瞬间即可打字                    |
| **关闭后清空输入**   | 下次打开是空白框，不是上次内容      |
| **最小化到托盘**     | 点 X 不是退出，是最小化到托盘       |

### 最小可用版本

```
呼出 → 聚焦 → 输入 → Enter提交 → 关闭
```


## 数据保留

- **logs/**：原始记录永久保留，用户自行管理
- **reports/**：生成的报告，按需保留
